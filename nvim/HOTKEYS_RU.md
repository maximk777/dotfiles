# Горячие клавиши Neovim

> Leader key: `<Space>`

## Тестирование (`<leader>t`)

| Клавиши | Действие |
|---------|----------|
| `<leader>tt` | Запустить ближайший тест |
| `<leader>tf` | Запустить все тесты в файле |
| `<leader>ta` | Запустить все тесты в проекте |
| `<leader>tl` | Повторить последний тест |
| `<leader>tr` | Запустить с флагом `-race` (Go) |
| `<leader>td` | Отладка ближайшего теста |
| `<leader>te` | Запустить с переменными окружения |
| `<leader>tE` | Запустить с доп. аргументами |
| `<leader>ts` | Показать/скрыть summary |
| `<leader>to` | Показать/скрыть output |

### Coverage

| Клавиши | Действие |
|---------|----------|
| `<leader>tC` | Загрузить coverage |
| `<leader>tS` | Показать summary coverage |
| `<leader>tH` | Переключить подсветку coverage |

|---------|-------|----------|
| `<leader>rr` | n, x | Меню рефакторинга |

## Юзкейсы

### Разработка на Go

1. Создать файл `.go` - автоматически добавится `package`
2. `<leader>tt` - запустить тест под курсором
3. `<leader>tr` - запустить с race detector
4. `<leader>te` - запустить с env переменными
5. `<leader>td` - отладка теста
6. `<leader>tC` - посмотреть coverage

### Отладка

1. `<leader>db` - поставить breakpoint
2. `<leader>dB` - условный breakpoint (например `i > 10`)
3. `<leader>dd` - запустить отладку
4. `<leader>dn/di/do` - пошаговое выполнение
5. `<leader>de` - посмотреть значение переменной
6. `<leader>dt` - завершить

### Навигация по проекту

1. `<leader>H` - добавить важные файлы в Harpoon
2. `<leader>1-5` - быстрое переключение между ними
3. `s` + 2 символа - прыжок к любому месту на экране
4. `<leader>cs` - outline функций в файле

### Рефакторинг

1. Выделить код в visual mode
2. `<leader>re` - извлечь в функцию
3. `<leader>rv` - извлечь в переменную
4. `<leader>ri` - inline переменную обратно
